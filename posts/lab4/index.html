Lab4: Roll-a-Ball in VR

adapt roll-a-ball into VR

Set up environment

  • create a new 3d project
  • File -> Build Profiles -> Android -> Switch Platform. Because Oculus we use is running on Android.
  • Import Oculus Integration to Unity, Though Oculus Integration is shown deprecated in Assetstore in 2024. To be consistent with the tutorials I follow, I didn’t use All in One SDK, which is the updated one. There is another reason: I don’t have OVR prefabs downloaded in Assets Folder with All in One SDK.
  • You may need to import Oculus XR Plugin for some version.
  • Project Setting -> XR Plug-In Management, check the Oculus for both the Android and PC.

Reuse Roll a ball

Following the tutorial provided in the course, I export the Roll-A-Ball game as a .unitypackage file by Assets -> Export Pacakge. There is something to tweak:

Then prepare the scene for the game.

  • Create a huge plane as the floor
  • a cube as a table
  • put them as children of an Empty Object named Environment
  • remove Main Camera and use the Assets -> Oculus -> VR -> Prefabs -> OVRCameraRig instead. Set the tracking to Floor Level and the Color Gamut to Quest.
    • tips: you may find “convert Main Camera” or something like that to conver Main Camera to OVRCameraRig if you use All-in-one SDK. I didn’t test it but I believe it should work.
  • add OVRControllerPrefab for both HandAnchors. LeftControllerAnchor for L Touch, RightControllerAnchor for R Touch.

Now we can import the .unitypackage file and put it on the table. some tips:

  1. put them as children of an empty object named roll-a-ball
  2. initialize empty object “roll-a-ball” position before binding old objects to it.
  3. rebake navMesh after resizing the ground if you want to keep the navMesh. for me, I only pick Player, Ground, Walls, Pickups.
  4. You can keep the previous UI! just need to select World Space in Canvas -> Render Mode. Remember to set reference back to our PlayerController script of roll-a-ball.

Add Colliders and Layers

  • create two layers “selection” for roll-a-ball gameobject and its all children.
  • add a box collider to the Ground and fit the size.
  • set Left/RightHandAnchor to selection layer too.
  • add a SphereCollider and adjust the radius to the same size of the controller prefab. Remember to check IsTigger
  • Edit -> Project Settings -> Physics, uncheck roll-a-ball layer with selection layer.
  • add rigid body on anchors and roll-a-ball if you don’t do that before. check kinematic for anchors and useGravity for Roll-a-ball.

using System.Collections;
using System.Collections.Generic;
using UnityEngine;

public class MySelct : MonoBehaviour
{
    public OVRInput.Controller controller;
    private float triggerValue;
    [SerializeField] private bool isInCollider;
    [SerializeField] private bool isSelected;
    private GameObject selectedObj;

    // Update is called once per frame
    void Update()
    {
        // Here we called the IndexTrigger value from controller,
        // so the Primary will map to right hand when the inspectoris RTouch in Unity.
        triggerValue = OVRInput.Get(OVRInput.Axis1D.PrimaryIndexTrigger, controller);

        if(isInCollider)
        {
            // not selected and pull the trigger
            if (!isSelected && triggerValue > 0.95f)
            {
                isSelected = true;
                selectedObj.transform.parent = this.transform;
                Rigidbody rb = selectedObj.GetComponent<Rigidbody>();
                rb.isKinematic = true;
                rb.useGravity = false;
                rb.velocity = Vector3.zero;
                rb.angularVelocity = Vector3.zero;
            }
            else if (isSelected && triggerValue <0.95f)
            {
                isSelected = false;
                selectedObj.transform.parent = null;
                Rigidbody rb = selectedObj.GetComponent<Rigidbody>();
                rb.isKinematic = false;
                rb.useGravity = true;
                rb.velocity = OVRInput.GetLocalControllerVelocity(controller);
                rb.angularVelocity = OVRInput.GetLocalControllerAngularVelocity(controller);
            }
        }
    }

    void OnTriggerEnter(Collider other)
    {
        if(other.gameObject.name == "roll-a-ball")
        {
            isInCollider = true;
            selectedObj = other.gameObject;
        }
    }

    void OnTriggerExit(Collider other)
    {
        if (other.gameObject.name == "roll-a-ball")
        {
            isInCollider = false;
            selectedObj = null;
        }
    }
}

then you can install the APK to the Oculus Quest headset. Remember to turn on developper mode.
